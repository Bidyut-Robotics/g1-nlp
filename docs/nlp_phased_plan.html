Phase 1 Core pipeline — voice in, voice out Weeks 1–4
ASR setup
Deploy Whisper large-v3 on Jetson Orin. Add VAD (Silero) to detect speech start/end. Benchmark WER on Telugu, Hindi, English samples.
Whisper, Silero VAD, Python, ONNX
Language identification
Route detected utterances to the correct downstream model. fastText lid.176 gives sub-10ms language detection.
fastText, langdetect
Basic intent classifier
Build intent taxonomy: greet, navigate, query, remind, handoff. Start with zero-shot GPT-4o; collect data to fine-tune later.
GPT-4o, LangChain, spaCy
TTS baseline
Integrate ElevenLabs (cloud) or Coqui TTS (on-device) for response speech. Test latency at Jetson Orin level for real-time feel.
ElevenLabs API, Coqui, pyttsx3
Deliverable: End-to-end demo — person speaks, robot responds in correct language. Latency <3s round-trip.
Phase 2 Understanding layer — entities, context, dialogue state Weeks 5–8
Entity extraction
Person names, room numbers, times, company-specific terms. Use spaCy with a custom NER model trained on office-domain data.
spaCy, Prodigy (annotation), custom NER
Dialogue manager
Build conversation state machine using LangGraph. Handle multi-turn flows like "take me to Room 5" → "which floor?" → walk action.
LangGraph, Rasa Core (optional), Redis
Context injector
Enrich every LLM prompt with: current time, known visitor identity, room the robot is in, last 3 conversation turns.
FastAPI middleware, Redis session store
Sentiment / affect detection
Classify frustration, urgency, formality. Adapt robot tone accordingly — more formal with executives, friendlier with guests.
XLM-RoBERTa, HuggingFace Transformers
Deliverable: Multi-turn dialogue that retains context across 5+ turns. Entity extraction F1 >0.85 on office domain test set.
Phase 3 Reasoning layer — RAG, tool use, action dispatch Weeks 9–14
RAG knowledge base
Index company FAQ, visitor directory, room map, leadership profiles into Chroma. Build retrieval pipeline with re-ranking.
LlamaIndex, Chroma, OpenAI embeddings, cross-encoder re-ranker
LLM tool calling
Define function schemas for calendar lookup, visitor check-in, weather fetch, room status. LLM decides when to call which tool.
GPT-4o function calling, FastAPI tool gateway
Action dispatcher
Translate NLP output into structured action commands for the robotics team API: navigate(room_id), gesture(wave), fetch(item).
Pydantic action schemas, ROS 2 bridge, REST API
Safety + guardrails
Add output filtering: PII scrubbing, topic boundary enforcement, hallucination confidence thresholds before response is spoken.
Guardrails AI, Llama Guard, custom rule layer
Deliverable: Robot can answer "Where is Dr. Kumar's meeting?" by fetching calendar + greeting the visitor by name. Action dispatch integration verified with robotics team.
Phase 4 Observability, eval loop, and hardening Weeks 15–18
Interaction logging
Log every ASR transcript, intent, entity, action, and response to TimescaleDB. Build Grafana dashboard for real-time monitoring.
TimescaleDB, Grafana, Kafka
Eval pipeline
Track WER, intent classification F1, response BLEU/ROUGE, action accuracy, p95 latency. Run weekly regression tests.
DeepEval, pytest, MLflow
Human-in-the-loop annotation
Flag low-confidence interactions for human review. Annotated data feeds fine-tuning of intent classifier and NER model.
Label Studio, Prodigy, LoRA fine-tuning
Latency optimization
Profile end-to-end: ASR, LLM inference, TTS. Quantize Whisper to int8. Use streaming TTS so robot starts speaking before full response is generated.
ONNX Runtime, TensorRT, streaming TTS
Deliverable: Production-ready NLP module. Intent F1 >0.92, end-to-end latency <2s, full observability dashboard live.
